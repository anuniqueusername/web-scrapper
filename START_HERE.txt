🚀 COMMON TASKS:

Change Scraping Interval:
  → Dashboard → Scheduling → ⚙️ Configure → Select interval → Save

Enable/Disable Scraper:
  → Dashboard → Scheduling → Toggle switch

Setup Discord Notifications:
  → Dashboard → Scheduling → Configure → Enable Discord → Paste URL → Save

View Listings:
  → Dashboard → Filter Results → Listings table

Run Scraper Now:
  → Dashboard → Scheduling → ▶️ Run Now button

View Status:
  → Dashboard → Status Dashboard (updates every 2 seconds)


⚙️ HOW IT WORKS:

Scraper:
  • Runs on interval (30s to 1 day)
  • Reads configuration from scraper-config.json
  • Updates status in scraper-status.json
  • Respects enabled/disabled state
  • Sends Discord/Slack notifications (if configured)

Dashboard:
  • Save config → Scraper picks it up
  • Auto-refreshes status every 2 seconds
  • Shows real-time running status
  • Displays new listings count
  • Tracks errors


📁 FILE STRUCTURE:

scraper-config.json (auto-created)
  → Your configuration (can edit manually)

scraper-status.json (auto-created)
  → Runtime status (updated by scraper)

data/listings.json (existing)
  → All scraped listings

app/
  → Next.js dashboard

components/
  → Dashboard UI components

package.json
  → Dependencies and scripts

• No need to restart scraper for config changes
• Errors are tracked in scraper-status.json


❓ TROUBLESHOOTING:

"Scraper disabled" in console?
  → Enable it: Dashboard → Scheduling → Toggle switch

No listings showing?
  → Wait for first scrape to complete (~15 seconds)
  → Check scraper console for messages

Dashboard not refreshing?
  → Check scraper is still running
  → Try refreshing browser
  → Check localhost:3000 in console

Webhooks not sending?
  → Verify URL is correct (copy/paste carefully)
  → Enable notifications in dashboard
  → Trigger a manual run
